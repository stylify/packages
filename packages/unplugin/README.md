## @stylify/unplugin introduction

Stylify is a library that uses CSS-like selectors to generate optimized utility-first CSS dynamically based on what you write.<br>
Don't study framework. Focus on coding.

Stylify Unplugin is universal plugin that integrates Stylify into Rollup, Webpack and Vite.

## ‚ö° Why Stylify instead of CSS or inline styles?
Because of [fewer CSS headaches](https://stylifycss.com/docs/get-started/why-stylify-css#problems-stylify-css-tries-to-solve), [faster coding](https://stylifycss.com/docs/get-started/why-stylify-css#faster-coding) and [extremely optimized output](https://stylifycss.com/docs/get-started/why-stylify-css#output-optimization).

## ‚ú® Features

- ‚úÖ [Syntax you already know](https://stylifycss.com/#zero-learning-curve). Don't waste time studying CSS framework.
- ‚úÖ Less switching between HTML/CSS files
- ‚úÖ [Automagic and Extremely tunned CSS optimization](https://stylifycss.com/#optimization)
- ‚úÖ [Simple CSS Bundles splitting](https://stylifycss.com/#bundles-splitting) for Layouts/Pages
- ‚úÖ [Easily Configurable and Extensible](https://stylifycss.com/#configuration)
- ‚úÖ [Prepared components](https://stylifycss.com/#components)
- ‚úÖ Define [Variables](https://stylifycss.com/docs/stylify/compiler#variables), [Components](https://stylifycss.com/docs/stylify/compiler#components), [Custom selectors](https://stylifycss.com/docs/stylify/compiler#customselectors)
- ‚úÖ [Add custom macros](https://stylifycss.com/docs/stylify/compiler#macros) like ml:2
- ‚úÖ Variables can be injected into CSS as CSS variables
- ‚úÖ CSS [variables can differ for each screen](https://stylifycss.com/docs/stylify/compiler#variables)
- ‚úÖ Simplify coding with [helpers](https://stylifycss.com/docs/stylify/compiler#helpers) like `color:lighten(#000,10)`
- ‚úÖ Style any device with dynamic [screens](https://stylifycss.com/docs/stylify/compiler#screens)
- ‚úÖ You can mark areas for which CSS should not be
- ‚úÖ Split bundles for page/layout/component
- ‚úÖ Selectors are minified from long `.color:blue` to short `.a`
- ‚úÖ No purge needed. CSS is generated only when something is matched
- ‚úÖ Components & Custom selectors are attached to utilities. No duplicated `property:value`
- ‚úÖ Hooks can modify CSS or output for example wrap it in CSS layers
- ‚úÖ Mangled (hidden/unreadable) HTML classes in production (if mangled)
- ‚úÖ [Try it with frameworks](https://stylifycss.com/docs/integrations) like Next.js, Astro.build, SolidJS, Qwik Symfony, Nette, Laravel
- ‚úÖ Works with bundlers like Webpack, Rollup, Vite.js
- ‚úÖ Generated CSS can be used within SCSS, Less, Stylus
- ‚úÖ CSS variables can be exported into external files and reused
